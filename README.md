## 🎯 Our Aim
Phone scams and social engineering attacks are becoming increasingly sophisticated. **KavachAI** aims to protect vulnerable users by actively listening to their phone calls in real-time and alerting them immediately if suspicious behavior, urgency-inducing language, or known scam keywords are detected.

---

## 🏗️ Core Architecture & Tech Stack

KavachAI is built using a modern decoupled architecture:

### 📱 1. Android Application (Client)
- **Language**: Java
- **Core Features**: 
  - `BroadcastReceiver` to detect incoming/outgoing calls in the background.
  - Foreground Service to maintain microphone access.
  - **Acoustic Coupling**: Forces the phone to Speakerphone mode to record both sides of the conversation via the device's microphone.
  - Real-time `WebSocket` streaming of raw PCM audio chunks.
  - Floating UI alerts triggered via broadcasts.

### 🧠 2. Python Server (Backend)
- **Language**: Python 3.10+
- **Core Features**:
  - `websockets` asynchronous server to receive audio streams from the Android app.
  - Integration with the **Sarvam AI Streaming STT API** for ultra-low latency Speech-to-Text in Indic languages / Code-Mixed English.
  - **Scam Detection Engine**: A rule-based risk scoring system that analyzes the live transcript for urgency markers (e.g., "OTP", "bank", "block", "police").
  - Streams continuous Risk Scores and Alerts back to the Android client.

---

## 🚧 Challenges & Current Limitations

### 1. Android Audio Capture Restrictions
Starting with Android 10, the OS strictly prohibits third-party apps from recording the `VOICE_CALL` (in-call downlink) audio stream to protect user privacy. 
- **Workaround used**: *Acoustic Coupling*. The app automatically turns on Speakerphone when a call connects, and uses the `VOICE_RECOGNITION` (Microphone) audio source to physically pick up the caller's voice from the device speaker.
- **Limitation**: This requires the user to hold the phone away from their ear or use the speakerphone in quiet environments. It struggles in extremely noisy places or if headphones are connected.

### 2. Speech-to-Text Latency
- The system relies on an active internet connection to stream audio to the Sarvam AI cloud. Flaky mobile data connections can delay transcripts and alerts.

## 🚀 Future Improvements

1. **On-Device Machine Learning (Edge AI)**:
   - Transitioning the NLP scam detection and STT models to run entirely on-device (e.g., using TFLite or ExecuTorch). This would eliminate the need for a backend server, reduce latency to zero, and guarantee absolute privacy since no audio would ever leave the phone.
2. **Advanced NLP Models**:
   - Replace the current keyword/regex-based scoring engine with a lightweight LLM (like Llama-3-8B or Gemma 2B) fine-tuned specifically on phishing and scam call transcripts to understand context and intent better.
3. **Accessibility Integration (Alternative to Acoustic Coupling)**:
   - Explore using Android Accessibility Services to read live closed-captions generated by the OS (Live Caption) as an alternative way to get transcripts without recording audio.

## 🛠️ Setup & Installation

### Backend Setup
1. Clone the repository and navigate to the `backend/` directory.
2. Create and activate a virtual environment:
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows: .\venv\Scripts\activate
   ```
3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
4. Create a `.env` file and add your Sarvam AI key:
   ```env
   SARVAM_API_KEY=your_api_key_here
   ```
5. Start the WebSocket server:
   ```bash
   python kavach_backend_server.py
   ```

### Android Setup (Local USB Testing)
1. Ensure your Android device (Android 10+) is connected via USB with Debugging enabled.
2. Port-forward the WebSocket connection to your local machine:
   ```bash
   adb reverse tcp:8765 tcp:8765
   ```
3. Open the `mobile_app` folder in Android Studio and hit **Run**, OR use gradle:
   ```bash
   ./gradlew installDebug
   ```
4. **Important**: Go to Android Settings -> Apps -> KavachAI -> Permissions and manually grant the **Phone** permission (required for background call detection).
